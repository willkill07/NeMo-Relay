A plugin's complete report on how it handled an intent bundle.

Every intent in the input bundle MUST have a corresponding outcome in the report. This is the critical observability contract per the design doc.

## Fields

### `request_id: Uuid`

ID of the request this report pertains to.

### `plugin_id: String`

Identity of the plugin that produced this report.

### `outcomes: Vec<IntentOutcome>`

Per-intent outcomes.

### `created_at: DateTime<Utc>`

When this report was generated.

Create a report where all intents are ignored (passthrough behavior).

Generates one [`IntentOutcome`](/reference/api/rust-library-reference/nemo-relay-adaptive/acg/types/struct-intentoutcome) per intent in the bundle, each with [`TranslationStatus::Ignored`](/reference/api/rust-library-reference/nemo-relay-adaptive/acg/types/enum-translationstatus) and the given reason code. This is the standard helper for passthrough and default plugin implementations.
