Capability registry tracking per-backend and per-model-family supported features.

The registry provides feature discovery so the policy engine knows which optimization intents can be expressed on which backend/model combinations.

## Two-Level Feature Lookup

[`BackendCapabilities`](/reference/api/rust-library-reference/nemo-relay-adaptive/acg/capability/struct-backendcapabilities) stores backend-level defaults plus per-model-family overrides via [`ModelFamilyCapabilities`](/reference/api/rust-library-reference/nemo-relay-adaptive/acg/capability/struct-modelfamilycapabilities). Feature lookups check the model family first, falling back to backend-level if the family is not registered.

## Built-in Defaults

[`CapabilityRegistry::with_defaults()`](/reference/api/rust-library-reference/nemo-relay-adaptive/acg/capability/struct-capabilityregistry) returns a registry pre-populated with known Anthropic and OpenAI capabilities.
